- 基础 URL：https://poloai.top
- 支持模型：Sora-2、豆包 Seedream、Veo、可灵、阿里万相、Flux 等
- 文本生成：Claude 模型 via models.json

### 三个开源项目评估

#### huobao-drama（推荐主体）
- 路径：`/root/huobao-drama/`
- 技术栈：Go + Vue3，SQLite，DDD 分层架构
- 完整链路：Drama → Episode → Storyboard → ImageGen → VideoGen → VideoMerge
- 三大素材体系：CharacterLibrary（角色库）、Scene（场景）、Prop（道具）
- Timeline 模型：Track → Clip → Transition → Effect（类剪映时间线）
- 视频合并：ffmpeg 片段拼接，支持多 provider（doubao/sora）
- 关键优势：Timeline 数据结构已接近剪映工程文件格式

#### Toonflow-app
- 路径：`/root/Toonflow-app/`
- 技术栈：Node.js + TypeScript，Vercel AI SDK
- AI Agent 架构：outlineScript Agent + storyboard Agent（segmentAgent → shotAgent）
- 流程：novel → outline → assets → storyboard → video
- 优势：Agent 分镜拆解逻辑好
- 劣势：无 Timeline/导出能力

#### Toonflow-web
- 前端项目，配合 Toonflow-app，参考价值有限

### 技术方案决策

#### 核心差异化：Capcut/剪映工程文件导出
- 三个项目都没做这个功能
- huobao-drama 的 Timeline 模型与剪映 draft_content.json 结构高度对应：
  - Timeline ↔ draft_content
  - TimelineTrack ↔ tracks[]
  - TimelineClip ↔ segments[]
  - ClipTransition ↔ transitions[]
  - ClipEffect ↔ effects[]
  - Duration/FPS/Resolution ↔ canvas_config

#### 方案：基于 huobao-drama 二次开发
1. 接入 PoloAPI 的 Sora/豆包/Veo 视频生成接口
2. 开发 Timeline → Capcut 工程文件导出模块（核心壁垒）
3. 借鉴 Toonflow 的 Agent 分镜拆解逻辑，增强剧本→分镜自动化

#### 产品定位
AI 生成短剧 + 一键导出到剪映二次编辑，打通"AI 生成"和"人工精修"的最后一公里
