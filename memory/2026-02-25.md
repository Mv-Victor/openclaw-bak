## moyin-creator（魔因漫创）竞品分析

### 项目信息
- GitHub: MemeCalculate/moyin-creator，1.1k stars
- 定位：AI影视生产级工具，剧本到成片全流程批量化
- 技术栈：Electron 30 + React 18 + TypeScript + Zustand 5 + Radix UI + Tailwind CSS 4
- 许可：AGPL-3.0（双重许可，商用需联系）
- 本地路径：`/root/moyin-creator/`（已 clone 但代码未 checkout，需从 upstream 拉取）

### 五大板块
📝 剧本解析 → 🎭 角色一致性 → 🌄 场景生成 → 🎬 导演分镜 → ⭐ S级(Seedance 2.0)

### 核心能力分析
1. **Character Bible（角色圣经）**：visualTraits + styleTokens + colorPalette + referenceImages + threeViewImages，6层身份锚点
2. **Prompt Compiler**：Mustache模板引擎，scene/character/camera/style分层组装
3. **S级板块（Seedance 2.0）**：
   - ShotGroup 多镜头合并叙事
   - @Image/@Video/@Audio 多模态引用
   - 首帧图网格拼接（N×N策略）
   - 动作+镜头语言+唇形三层融合
   - 组级AI校准：narrativeArc + transitions + groupAudioDesign
   - 视频延长/编辑能力
4. **Worker Protocol**：主线程↔Worker 通信协议，支持剧本生成/场景执行/重试/取消
5. **批量化队列**：并行生成 + 自动重试

### 与 huobao-drama 对比
我们有他们没有的：镜头语法规则库、情绪编排引擎（EmotionArc+SilenceWindow）、Motion Prompt三层结构
他们有我们缺的：角色一致性系统、Prompt模板引擎、多镜头合并叙事（ShotGroup）、Seedance 2.0集成、批量化队列

### 建议行动
1. 优先移植角色一致性系统到 huobao-drama
2. 白骨夫人项目中实践 ShotGroup 概念
