# 2026-02-25 工作记录

## 情绪编排引擎 + 镜头语法库 — 技术方案（已确认）

### 背景
- 栋少提出：当前AI短剧的瓶颈在于缺乏长镜头、怼脸大镜头、情绪渲染BGM的精准卡点能力
- 举例：夏洛特烦恼中袁华出场BGM《一剪梅》片段 — 镜头语言+BGM卡点+情绪反差三层同时工作
- 栋少确认方向值得深挖，要求：先不做timeline转剪映工程文件，但要考虑兼容性

### 决策
- 在 huobao-drama 的 Timeline 中增加情绪编排引擎和镜头语法库
- 把 Toonflow 的 motion prompt 迁移过来
- 先在 huobao-drama 上验证有情绪节奏的短剧生产思路

### 三层架构设计
1. **Layer 1: 镜头语法库 (Shot Grammar Library)**
   - 镜头语法规则引擎（景别/角度/运镜 → 情绪/节奏映射）
   - Motion Prompt 生成（从 Toonflow 迁移模板）
   - 新文件: `domain/models/shot_grammar.go`, `application/services/shot_grammar_service.go`

2. **Layer 2: 情绪编排引擎 (Emotion Arc Engine)**
   - 跨镜头情绪弧线分析（EmotionBeat: emotion/intensity/tension/resolution）
   - BGM Cue 标注（intro/build/climax/fade/silence）
   - 节奏调整（pace_target: slow/medium/fast/burst）
   - 新文件: `domain/models/emotion_arc.go`, `application/services/emotion_arc_service.go`

3. **Layer 3: Timeline Orchestrator（先定接口，暂不实现）**
   - BGM音轨编排、速度曲线、转场控制
   - 预留 Capcut draft_content.json 兼容

### 现有代码改造点
- `storyboard_service.go`: 分镜拆解后自动调用情绪弧线分析 + 镜头语法校验
- `drama.go` Storyboard 模型: 新增 MotionPrompt, EmotionIntensity, Tension, BgmCue, PaceTarget 字段（nullable）
- `frame_prompt_service.go`: generateVideoPrompt() 改用 Motion Prompt + 镜头语法规则
- `video_merge_service.go`: 暂不改，预留接口

### 实施节奏
- Phase 1（本周）: 镜头语法库 + Motion Prompt 迁移
- Phase 2（下周）: 情绪编排引擎
- Phase 3（后续）: Timeline 编排 + Capcut 兼容

### 代码现状摸底
- huobao-drama Timeline 结构: Timeline → Track(video/audio/text) → Clip → Effect/Transition
- Storyboard 已有字段: Emotion, BgmPrompt, SoundEffect, ShotType, Angle, Movement（但未被消费）
- Toonflow motion prompt: 完整的镜头语言规范（画面类型/镜头运动/角色动作/环境动态/速度节奏/氛围风格），输出 {time, name, content} JSON
- video_merge_service: 只做 FFmpeg 简单拼接，无音画同步
