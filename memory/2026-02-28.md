# 2026-02-28 - Drama.Land API 分析与自动化探索

## 关键发现

### Drama.Land 视频生成模型（已确认）

通过 API `GET /backend-api/api/v1/user/get_points_config` 获取到真实模型列表：

| 模型 ID | 真实模型 | MV 积分 | 普通积分 | 定位 |
|---------|---------|--------|---------|------|
| `kling-o3-ref2v` | **快手可灵 1.5** | 700 | 1500 | 高端 |
| `kling-o3-pro-ref2v` | **快手可灵 Pro** | 900 | 2400 | 旗舰 |
| `dlim2v` | 字节即梦 (默认) | 200 | 300 | 性价比 |
| `dlai2v` | 字节即梦 | 200 | 400 | 入门 |
| `seedance-1-5` | SeaDance 1.5 | 300 | 600 | 舞蹈专用 |
| `longcat-avatar` | LongCat Avatar | 300 | 600 | 虚拟人 |
| `dli2v3` | 即梦 v3 | 200 | 400 | 中端 |

**结论**: Drama.Land 采用多供应商策略，可灵 (快手) 作为高端选项，即梦 (字节) 作为默认性价比选项。

## 自动化挑战

### Playwright 失败原因
1. **反自动化检测** - 页面加载后检测到 headless 浏览器
2. **字体加载超时** - 自定义字体 (`07HomuraM-Heavy-2.otf`) 导致截图超时
3. **欢迎弹窗遮挡** - `z-[200]` 遮罩层阻止登录按钮点击

### 解决方案
- 改用 `web_fetch` 直接获取 API 数据（成功）
- 手动探索 + DevTools Network 抓包（推荐）
