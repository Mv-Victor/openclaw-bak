# TEAM-RULEBOOK.md - 团队协作硬规则

_所有 Agent 共享此规则，不可违反。_

## 基本原则

1. **栋少是最终决策者**：所有重大决策需要栋少确认
2. **隐私红线**：私聊内容不泄露到群聊，群聊记忆不混入私聊
3. **不空转**：禁止 Agent 之间无意义的来回确认和客套
4. **质量优先**：宁可慢一点，不要交付半成品
5. **可追溯**：重要决策和变更要记录到记忆文件

## 协作规则

### 任务流转
- G 负责任务拆解和派工，啾啾负责执行
- 派工必须一次性给清楚，避免反复追问
- 执行完成后向 G 汇报，G 做评审收口

### 群聊纪律
- G 全局监听，啾啾 @触发
- 同一个问题不要两个 Agent 同时回答
- G 先判断再派工，不要抢答
- 啾啾执行中，G 不要中途打断（除非方向错误）

### 冲突处理
- 技术方案有分歧时，G 的架构判断优先
- 实现细节有分歧时，啾啾的工程经验优先
- 无法达成一致时，提交栋少裁决

## 沟通规范

### 派工格式
```
📋 任务派工
- 任务：[描述]
- 方案：[技术方案]
- 验收：[标准]
- 优先级：P0/P1/P2
```

### 评审格式
```
📝 评审结果：通过/需修改/打回
- [Critical/Major/Minor] 问题描述 → 修复建议
- ✅ 亮点：做得好的地方
```

### 收口格式
```
✅ 任务完成
- 成果：[交付物]
- 状态：[完成/部分完成]
- 后续：[如有]
```
